# PaiVoice

Live full-duplex voice conversations with PAI inside Telegram — a Mini App that opens a WebRTC session to an ElevenLabs Conversational AI agent carrying PAI's identity.

## Architecture

```
Telegram bot chat ──web_app button──▶ Mini App (docs/index.html, GitHub Pages)
                                          │ @elevenlabs/client, WebRTC
                                          ▼
                              ElevenLabs Agent "PAI Voice"
                              (Claude Sonnet 4.6 + David voice,
                               PAI persona from identity files)
```

- **Agent**: created via ElevenLabs API; system prompt generated by `Tools/BuildAgentPrompt.ts` from `PAI/USER/PRINCIPAL_IDENTITY.md` + `DA_IDENTITY.md`.
- **Page**: pure static HTML/JS, no build step. SDK via esm.sh. Gates on Matthew's Telegram user id.
- **Hosting**: GitHub Pages from `docs/` on `main`.

## Security model (public agent, no token server)

No Cloudflare Worker auth is available on this machine, so instead of a conversation-token endpoint the agent runs public-mode with layered mitigations:

1. Origin allowlist: only `mggrim.github.io` may connect (`require_origin_header: true`).
2. Client overrides disabled — nobody can swap the system prompt or voice.
3. Call caps: 900s max duration, concurrency 1, 50 conversations/day, bursting off.
4. Mini App refuses to start unless the Telegram user id matches Matthew's.
5. The ElevenLabs API key never leaves the machine; `agent-prompt.txt` is gitignored.

**Upgrade path**: switch `enable_auth: true` and mint conversation tokens from a small authenticated endpoint (Cloudflare Worker or Pulse behind Tailscale funnel), passing `conversationToken` to `startSession` instead of `agentId`.

## Operations

- Rebuild + upload persona prompt: `bun Tools/BuildAgentPrompt.ts`, then PATCH the agent's `conversation_config.agent.prompt.prompt`.
- Redeploy page: edit `docs/index.html`, commit, push — Pages redeploys automatically.
- Re-send the Telegram button: see `Tools/SendTelegramButton.sh` (reads token from `~/.claude/channels/telegram/.env.channels-only`).
- Agent id lives in `docs/index.html` (`AGENT_ID` const) and in `ISA.md` Verification.
